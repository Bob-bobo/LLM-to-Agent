## 1. 产品概述
### 1.1 产品背景
BobPet 是一款面向开发者、设计师及其他电脑深度用户的 Windows 11 桌面伴侣插件。它通过一个常驻桌面的可爱宠物形象，提供轻量级 AI 对话陪伴，并能感知用户的工作状态，给予相应的情感化反馈。产品旨在缓解长时间工作带来的孤独感，提升工作效率与情绪体验。

### 1.2 目标用户
 interviews-ui
- **开发者**：长时间使用 IDE（如 Cursor、Claude Code、Codex）进行编码，需要间歇性放松与陪伴。
- **设计师/产品经理**：频繁使用设计工具，工作过程中需要灵感激发或简单交流。
- **学生/研究人员**：在学习和研究过程中希望有个“伙伴”在一旁，增加学习乐趣。
- **任何希望桌面更有生气的电脑用户**。

### 1.3 核心价值主张
1. **零配置启动**：首次运行自动完成环境检测与模型配置，用户无需关心技术细节。
2. **本地优先**：日常对话完全在本地完成，保护隐私，降低延迟。
3. **情景感知**：智能感知用户工作状态，并做出相应反应，让桌宠“活”起来。
4. **高度可定制**：支持不同人格适配器与模型接入方式，满足个性化需求。

### 2.2 详细功能说明
#### 2.2.1 首次启动向导流程
1. 用户双击 BobPet.exe 启动应用。
2. 应用自动检测系统环境：
   - 检查 GPU 是否可用（CUDA/Metal/DirectML）。
   - 检查内存大小是否满足本地模型最低要求（如 8GB）。
   - 检查是否已安装必要运行时（Python 3.8+，可选）。
3. 弹出引导对话框，提供两个选项：
   - **使用本地模型**：提示用户选择本地模型文件路径（或提供默认 MiniCPM 下载指引）。
   - **接入线上服务**：进入配置表单，填写 API Base URL、API Key 等信息。
4. 根据用户选择，自动下载必要依赖或测试连接，完成后进入主界面。

#### 2.2.2 悬浮聊天交互流程
1. 用户按 `Ctrl+Shift+M`，在宠物附近弹出聊天气泡（初始位置可配置）。
2. 气泡下部为输入框，上部为对话历史区域。
3. 用户输入问题后，按 `Enter` 发送。
4. 根据配置的模型，请求本地或云端推理，回复内容流式输出到气泡。
5. 若启用思考模式（`Ctrl+Shift+T`），则同时显示模型内部的“思考过程”（如 chain-of-thought）。
6. 用户可按 `Esc` 关闭气泡，或点击气泡外区域关闭。

#### 2.2.3 工作状态感知规则
| 监测目标 | 活动判定条件 | 宠物状态 | 表现行为 |
|----------|---------------|----------|----------|
| Cursor/Claude Code/Codex 等 IDE | 进程存在且窗口处于激活状态，且有持续的键盘输入或文件保存事件 | **工作** | 宠物做出打字、翻书等动画，表情专注 |
| 同上 | 进程激活但无输入超过 30 秒 | **思考** | 宠物托腮、转圈、头上冒出问号气泡 |
| 编译/运行成功（通过监控输出日志关键词） | 检测到 "Build succeeded"、"Compilation finished" 等 | **完成** | 宠物跳跃、撒花、播放庆祝音效（可选） |
| 长时间无用户交互（全局） | 系统无键盘鼠标操作超过 5 分钟 | **等待关注** | 宠物打哈欠、趴下睡觉，偶尔醒来看看屏幕 |
| 所有监控的 IDE 进程均关闭 | 无相关进程运行 | **休息** | 宠物玩耍、悠闲走动、发呆 |

## 3. 非功能性需求
### 3.1 性能
- **启动时间**：首次启动（含环境检测）≤ 30 秒；后续启动 ≤ 5 秒。
- **对话响应延迟**：本地模型首次推理 ≤ 3 秒，后续 ≤ 1 秒；云端 API 请求 ≤ 2 秒（受网络影响）。
- **资源占用**：空闲时内存占用 ≤ 200 MB，CPU 占用 ≤ 2%；对话时内存增量 ≤ 500 MB。
- **动画流畅度**：宠物动画 ≥ 30 FPS，不影响主机 UI 流畅度。

### 3.2 兼容性
- **操作系统**：Windows 11 (22H2 及以上)，未来可考虑 Windows 10。
- **硬件**：支持集成显卡与独立显卡（CUDA 11.8+ / DirectML / Metal for future macOS）。
- **IDE 监控**：支持 Cursor、VS Code（Claude Code 插件）、Codex 等常见开发工具；可通过进程名与窗口标题正则匹配扩展。

### 3.3 安全性
- **本地数据**：所有对话历史、配置信息均存储在用户本地 AppData 目录，不上传。
- **API Key 存储**：若使用线上模型，API Key 以加密方式存储在本地的配置文件中。
- **模型安全**：本地模型运行在沙箱环境中，避免恶意模型代码执行。

### 3.4 可维护性与扩展性
- **插件架构**：核心（宠物渲染、状态机）与功能模块（聊天、状态感知）解耦，便于新增功能。
- **配置驱动**：人格适配器、状态映射规则均采用配置文件，无需修改代码即可扩展。
- **日志系统**：分级别（INFO/WARN/ERROR）日志，输出到文件，便于排查。

## 4. 技术架构建议
### 4.1 技术栈选型
| 组件 | 推荐技术 | 备注 |
|------|-----------|------|
| **桌面应用框架** | Electron + Vue 3 / React | 跨平台 UI，便于未来扩展 macOS/Linux。 |
| **宠物渲染** | Canvas 2D / CSS 动画 | 轻量级，性能好；也可考虑 Spine 骨骼动画。 |
| **本地模型推理** | Ollama / llama.cpp | 支持 MiniCPM 等常见 GGUF 格式，提供简单 API。 |
| **进程监控** | Node.js `ps` / `wmctrl` (Windows) | 通过轮询或事件监听获取活动窗口信息。 |
| **配置存储** | `electron-store` | 简单键值存储，支持加密字段。 |
| **HTTP 客户端** | `axios` | 调用云端 API。 |

### 4.2 模块划分
1. **主进程（Main Process）**：负责窗口管理、系统托盘、进程监控、配置读写。
2. **渲染进程（Renderer Process）**：
   - **宠物视图**：负责绘制宠物动画、处理拖拽。
   - **聊天视图**：悬浮气泡的 UI 与对话逻辑。
   - **设置视图**：多个配置页面的渲染与表单处理。
3. **本地模型服务（可选）**：一个子进程运行 Ollama，通过 REST API 与主进程通信。

### 4.3 数据流示意
```
用户输入
   ↓
[聊天气泡] → 请求发送 → [模型网关] → 本地模型 (Ollama) 或 云端 API
   ↑                                         ↓
对话历史 ← 流式接收 ← 响应返回 ← 推理结果
```

### 4.4 状态感知实现方案
- **轮询方式**：每 2 秒检查一次活动窗口进程与用户输入事件。
- **事件订阅**：使用 Windows API 监听窗口焦点变化、键盘/鼠标全局钩子（需注意性能与权限）。
- **规则引擎**：将状态判定规则定义为可配置的 JSON，便于调整。

## 5. 用户界面设计（概要）
### 5.1 桌面宠物
- 默认形象：卡通猫娘（neko），大小约 120x120 像素。
- 可拖动，拖动时显示半透明轮廓。
- 右键点击弹出快捷菜单（隐藏、设置、退出）。

### 5.2 悬浮气泡
- 设计类似聊天工具气泡，圆角，阴影。
- 可调整大小（右下角拖拽）。
- 对话历史支持滚动，新消息自动滚到底部。

### 5.3 设置窗口
- 侧边栏导航：模型设置、人格设置、快捷键、关于。
- 模型设置页：选项卡切换“本地模型”/“线上服务”，表单对应不同字段。
- 人格设置页：卡片式展示可用人格，右侧预览，底部“导入”按钮。
- 快捷键设置页：列表显示当前快捷键，可点击修改。

## 6. 开发里程碑建议
| 阶段 | 目标 | 预计工期 |
|------|------|----------|

## 7. 开放问题与未来迭代
- **多平台支持**：未来可适配 macOS、Linux。
- **宠物外观市场**：允许用户下载/购买不同宠物皮肤。
- **语音交互**：集成语音识别与合成，实现语音对话。
- **第三方插件**：开放 API，允许开发者编写扩展（如连接 GitHub、监控服务器状态）。
- **情感记忆**：让宠物能记住过往对话，形成长期陪伴感。
